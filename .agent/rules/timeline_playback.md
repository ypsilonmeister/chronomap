# タイムライン・プレイバックルール (timeline_playback.md)

本プロジェクトにおけるタイムスライダーUI およびプレイバック機能に関する制約事項です。

## 1. タイムスライダー UI
- キャンバス下部にシークバー形式のタイムスライダーを常時配置する。
- スライダーの左端は該当ページ内で最も古いノードの `createdAt` とし、右端は最も新しい `updatedAt` とする。
- スライダーのドラッグ操作に追従して、キャンバスの描画が即座に更新される（60fps を維持）。

## 2. 時刻指定によるスナップショット表示
- スライダーが指す時刻 T に基づき、キャンバスに描画するデータを以下のロジックでフィルタリングする:
  - **表示**: `createdAt <= T` であるノード・エッジを通常描画する。
  - **非表示または半透明**: `createdAt > T`（指定時刻より未来に作成された）ノード・エッジは非表示、または半透明で描画する。
  - **テキスト・写真の状態復元**: 各ノードの `updatedAt` が T 以降の場合、T 時点でのテキスト・写真の状態を可能な限り再現する（履歴データの保持方式に依存）。

## 3. 自動再生
- 再生ボタンを押すと、スライダーが左端（最古の `createdAt`）から右端（最新の `updatedAt`）まで自動的に進行する。
- 再生速度は以下の段階で切り替え可能にする:
  - **等倍速 (1x)**: 実際の経過時間と同じペースで再生。
  - **5倍速 (5x)**
  - **10倍速 (10x)**
- 再生中、マインドマップがゼロの状態から順次ノード・エッジが出現し「成長」していくアニメーションを実現する。
- 一時停止ボタンで任意の時点で停止可能にする。
