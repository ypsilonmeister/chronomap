# データ管理 & 同期ルール (data_sync.md)

本プロジェクトにおける IndexedDB の永続化、タイムスタンプ管理、および Google Drive 同期に関する制約事項です。

## 1. ローカルファースト & IndexedDB
- 原則としてすべての思考データ（Page, Node, Edge）はブラウザの IndexedDB に即時保存する。
- ライブラリ `idb` を用い、Promise ベースでトランザクションを非同期制御する。
- 完全にオフライン状態でも、データの作成、編集、閲覧、削除が支障なく行えるようにする。

## 2. タイムスタンプの厳格な管理
- すべてのデータ変更時に、以下の時間情報を ISO 8601 形式の文字列で記録する。
  - **Node.createdAt**: ノードが生成された瞬間（読み上げによるテキスト変換開始時ではなく、ノード領域が定義された時点）。
  - **Node.updatedAt**: テキスト編集の確定、写真添付の完了、ノード位置ドラッグ完了など、編集操作が確定した瞬間に更新する。
  - **Edge.createdAt**: 親子ノードが結線された瞬間。
- タイムライン機能で「過去の特定時刻」を表示する際、この `createdAt` および `updatedAt` のタイムスタンプを参照し、表示/非表示および半透明の判定を確実に行う。

## 3. 写真の添付と最適化
- 各ノードに画像を紐付ける際、IndexedDB のストレージ容量制限および同期パフォーマンス低下を防ぐため、以下の最適化を適用する。
  - 添付画像は、描画および転送用にブラウザ側でリサイズ（例: 最大幅 1024px に縮小）および圧縮（JPEG/WebP 等）を施してから IndexedDB に格納する。
  - IndexedDB 内ではバイナリ（Blob または ArrayBuffer）として保持し、レンダリング時に `URL.createObjectURL` で Blob URL を生成して Canvas に描画する。

## 4. Google Drive クラウド同期
- Google Drive API (OAuth2) を使用し、アプリ専用フォルダ（`appDataFolder`）にデータを JSON やバイナリ形式でバックアップ・復元する。
- **競合解消ポリシー**: ローカルデータとクラウドデータの差分が発生した場合は、各エンティティの `updatedAt` を比較し、常に更新日時が新しい方を優先してマージする。
