# キャンバスレンダリング & インタラクションルール (canvas_rendering.md)

本プロジェクトにおけるマインドマップ描画キャンバスの実装およびインタラクションに関する制約事項です。

## 1. 描画パフォーマンス
- **目標**: タイムライン移動時およびドラッグ・ズーム操作時の描画遅延を **16ms (60fps) 以内**に維持する。
- **実装方針**:
  - DOM 要素の増大によるオーバーヘッドを避けるため、HTML5 Canvas API を使用して一括描画する。
  - レンダリング時は、画面外にあるノードの描画スキップ（Culling）を実装する。
  - キャンバスの再描画は `requestAnimationFrame` を経由してバッチ処理し、不要なトリガー（連続的な重複呼び出し）を防ぐ。

## 2. ズーム & パン（操作）
- マウスクリックおよびドラッグによるキャンバスの自由移動（パン）をサポートする。
- マウスホイールによる拡大・縮小（ズーム）に対応する。ズームはカーソル位置を中心に拡大縮小するようアフィン変換を正しく行う。
- キャンバスのダブルクリック、または既存ノードの派生ボタン押下でノードを動的に追加できること。

## 3. キーボードショートカット
キャンバスフォーカス時に以下のショートカットを確実に動作させる。
- **Enter**: 選択中のノードと同一階層（兄弟）にノードを追加。
- **Tab**: 選択中のノードの下位階層（子）にノードを追加。
- **Delete / Backspace**: 選択中のノードおよびその配下の子ノードを一括削除（確認ダイアログを表示）。
- **F2**: 選択中のノードをインラインテキスト編集モードへ移行。
- **Ctrl + Z / Ctrl + Y**: キャンバス内の操作履歴の Undo（やり直し）/ Redo（元に戻す）。
