# ページ管理ルール (page_management.md)

本プロジェクトにおけるマインドマップページの一覧表示、検索、ライフサイクル管理に関する制約事項です。

## 1. ページ一覧表示
- 作成済みのマインドマップ（ノート）を一覧画面で表示する。
- ソート順として以下の 3 種類を切り替え可能にする:
  - 作成日時順 (`createdAt` 降順)
  - 最終更新日時順 (`updatedAt` 降順)
  - タイトル五十音順 (`title` 昇順)

## 2. 検索機能
- 検索窓にテキストを入力すると、以下を対象に全文検索を実行する:
  - ページタイトル (`Page.title`)
  - 全ページのノード内テキスト (`Node.text`) — 音声認識テキスト・キーボード入力テキスト双方を含む
- 検索は IndexedDB 上のローカルデータに対して行い、ネットワーク通信を発生させない。

## 3. ライフサイクル管理
- **新規作成**: ページ生成時に `pageId` (UUIDv4)、`createdAt`、`updatedAt` を自動付与する。
- **タイトル変更**: ページタイトルの編集時に `updatedAt` を更新する。
- **複製（クローン）**: 既存ページのすべてのノード・エッジを深コピーし、新しい `pageId` および新しいタイムスタンプ群を付与して新規ページとして保存する。
- **削除**: ページ削除時は配下のすべてのノード・エッジ・添付画像データも IndexedDB から完全に除去する。削除前に確認ダイアログを表示する。
