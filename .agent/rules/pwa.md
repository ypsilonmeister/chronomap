# PWA & オフライン要件ルール (pwa.md)

本プロジェクトにおける Progressive Web App (PWA) 対応とオフライン動作に関する制約事項です。

## 1. Service Worker
- `vite-plugin-pwa` を用いて Service Worker を自動生成し、静的アセット（HTML, CSS, JS, フォント, 画像）をプリキャッシュする。
- キャッシュ戦略は **Cache First** を基本とし、ネットワーク不到達時もアプリが完全に起動・動作すること。

## 2. オフライン動作の保証
- ネットワーク接続がない状態で以下の操作がすべて正常に行えること:
  - アプリの起動とページ一覧の表示
  - マインドマップの新規作成・編集・削除
  - 音声入力（Web Speech API のオフライン対応はブラウザ依存のため、非対応時はキーボード入力にフォールバック）
  - タイムライン・プレイバック
- Google Drive 同期のみオンライン必須とし、オフライン時は同期ボタンを無効化（グレーアウト）する。

## 3. インストール体験
- Web App Manifest (`manifest.json`) を配置し、ホーム画面への追加（A2HS）を有効にする。
- `display: "standalone"` を指定し、ブラウザ UI を非表示にしたネイティブアプリ風の体験を提供する。
