# メディア入力ルール (media_input.md)

本プロジェクトにおける音声入力（Speech-to-Text）および写真添付に関する制約事項です。

## 1. 音声入力 (Speech-to-Text)
- ノードをタップした際に表示されるポップアップメニューから「音声入力」を起動する。
- **使用 API**: Web Speech API (`SpeechRecognition` / `webkitSpeechRecognition`)。
- 音声認識はリアルタイムまたは発話終了時にテキストへ変換し、対象ノードの `text` 属性に自動格納する。
- テキスト格納確定時に `Node.updatedAt` を更新する。
- **ブラウザ互換性**: Web Speech API は Chrome, Safari, Edge で動作する。非対応ブラウザではフォールバックとしてキーボード入力を促すメッセージを表示する。

## 2. 写真添付
- 各ノードに対して画像を 1 枚以上紐付け可能にする。
- 画像取得元:
  - デバイスのカメラ起動（`<input type="file" accept="image/*" capture="environment">`）
  - ローカルストレージからのファイル選択
- 選択された画像はブラウザ側で適切なサイズにリサイズし（最大幅 1024px）、IndexedDB に Blob として保存する。
- キャンバス上では対象ノード内にサムネイル（縮小画像）を描画する。
- ノードをタップ/クリックすると拡大表示モーダルを展開し、元画像を閲覧可能にする。
