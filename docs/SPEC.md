# **ChronoMap 詳細仕様書**

Google Coding Agent (Antigravity) 開発インプット用要件定義資料

## **1\. システム概要**

本システムは、ユーザーが空間的かつ直感的に思考を記録・整理するためのローカルファーストなWebアプリケーションです。マインドマップによる視覚的UIをベースに、ノード単位での音声入力、キーボード入力、写真添付をシームレスに行うことができます。さらに、それらの思考データの変遷を時系列で追体験できる「タイムライン・プレイバック機能」を備え、思考のプロセスそのものを資産化することを目的とします。

## **2\. 機能要件**

### **2.1 ページ管理機能**

* **ページ一覧表示：** 作成済みのマインドマップ（ノート）を一覧表示する。作成日時順、最終更新日時順、タイトル順でのソートが可能。  
* **ページ内・全体検索：** 検索窓に入力された文字列を対象に、ページタイトルおよび全ページのノード内テキスト（音声認識テキスト・キーボード入力テキスト双方）を対象とする全文検索を実行する。  
* **ライフサイクル管理：** ページの新規作成、タイトル変更、複製（クローン）、削除機能。

### **2.2 マインドマップ・キャンバス機能**

* **無限キャンバス操作：** ドラッグ（またはタッチスワイプ）によるキャンバスの自由移動（パン）。マウスホイール（またはピンチイン・アウト）による拡大・縮小（ズーム）。  
* **ノードの操作：** キャンバスのダブルタップ、または特定ノードからの派生ボタン押下による新規ノード追加。ドラッグ＆ドロップによるノードの位置変更。選択したノードおよびその配下の子ノード一括削除。  
* **キーボードショートカット：**  
  * Enter: 選択中ノードと同一階層（兄弟）にノードを追加。  
  * Tab: 選択中ノードの下位階層（子）にノードを追加。  
  * Delete / Backspace: 選択中のノードを削除。  
  * F2: 選択中のノードをテキスト編集モードへ移行。  
  * Ctrl \+ Z / Ctrl \+ Y: 操作の取り消し（Undo）およびやり直し（Redo）。

### **2.3 メディア入力・メタデータ機能**

* **音声入力（Speech-to-Text）：** ノードをタップした際に表示されるポップアップメニューから「音声入力」を起動。音声認識処理を行い、リアルタイムまたは発話終了時にテキストへと変換し、ノードのテキスト属性に自動格納する。  
* **写真添付：** 各ノードに画像を紐付け。デバイスのカメラ起動、またはローカルストレージからの画像選択に対応。選択された画像は適切なサイズにリサイズされ、該当ノード内にサムネイルとして描画。ノードタップで拡大表示モーダルを展開。  
* **自動タイムスタンプ付与：**  
  * createdAt: ノードまたはエッジ（結線）が生成された瞬間のシステム時刻（ISO 8601形式）を自動記録。  
  * updatedAt: ノードのテキスト書き換え、写真追加、位置変更等が行われた瞬間に自動更新。

### **2.4 タイムライン・プレイバック機能**

* **タイムスライダーUI：** キャンバス下部に配置される時間制御シークバー。表示範囲は、該当ページ内の「最も古いノードの createdAt」から「最新の updatedAt」までをカバーする。  
* **プレイバック制御：** スライダーを手動でドラッグ移動することで、指定した時刻におけるマインドマップの状態（当時存在していたノード、結線、テキスト、写真）をキャンバス上に正確に再現。指定時刻より未来のデータは非表示または半透明化する。  
* **自動再生：** 再生ボタン押下により、可変速（等倍、5倍、10倍速など）でマインドマップがゼロから成長していくプロセスをアニメーションで実行する。

### **2.5 データ保存・同期機能**

* **ローカルファーストストレージ：** 原則としてすべてのデータ（テキスト、ノード座標、写真バッファなど）はブラウザの IndexedDB 内に永続化され、完全なオフライン環境での起動・閲覧・編集を保証する。  
* **クラウドストレージ同期（Google Drive連携）：** Google OAuth2 による認証を通じ、アプリ専用フォルダ（appDataFolder）を対象にローカルデータをJSON形式およびバイナリ群としてバックアップ。競合時は最終更新日時（updatedAt）が新しいデータを優先する。

## **3\. タイムラインのデータ生成・動作イメージ**

アプリ内部でどのように時間が記録され、マインドマップが構築されていくかのプロセスモデル：

| 時間 | 動作・内容   |
| :---- | :---- |
| **15:00:10** | **最初の発見（音声入力によるノード生成）** 中心ノードから最初の枝が伸びる。 *「きょうの公園で、すごいやつ見つけた！」* |
| **15:01:45** | **視覚情報の追加（写真添付）** 該当ノードに、対象（虫や植物、石など）の写真が添付され、ノード内にサムネイルがポップアップ。 |
| **15:02:30** | **ディテールの深掘り（連続音声入力）** 写真ノードからさらに2つの子ノードがハイスピードで派生。 *「あしがすごいいっぱいある」 「はっぱのうらにかくれた！」* |
| **15:07:15** | **考察と文字での補足（キーボード入力）** 少し時間が空いたあと、じっくり考えてからタイピングでノードを追加。 *「たぶんクモのなかま。あしたもしらべる」* |

## **4\. データモデル設計（スキーマ定義案）**

### **4.1 ページエンティティ (Page)**

{  
  "pageId": "UUIDv4 (string)",  
  "title": "string",  
  "createdAt": "string (ISO 8601)",  
  "updatedAt": "string (ISO 8601)",  
  "nodes": \[ ...NodeEntity \],  
  "edges": \[ ...EdgeEntity \]  
}

### **4.2 ノードエンティティ (Node)**

{  
  "id": "UUIDv4 (string)",  
  "text": "string",  
  "media": {  
    "hasImage": "boolean",  
    "imageRef": "string (Local Blob URL / Google Drive FileID)",  
    "hasAudio": "boolean",  
    "audioRef": "string (Local Blob URL / Google Drive FileID)"  
  },  
  "position": { "x": "number", "y": "number" },  
  "createdAt": "string (ISO 8601)",  
  "updatedAt": "string (ISO 8601)"  
}

### **4.3 エッジエンティティ (Edge)**

{  
  "id": "UUIDv4 (string)",  
  "source": "string (Node.id)",  
  "target": "string (Node.id)",  
  "createdAt": "string (ISO 8601)"  
}

## **5\. 非機能要件**

* **PWA対応：** Service Workerによる静的アセットのキャッシュを行い、スタンドアロンでのオフライン起動を可能にする。  
* **描画パフォーマンス：** 1ページあたり最大500ノード、画像添付50枚以上の環境下でも、タイムラインスライダー移動時の描画遅延が16ms（60fps）以内となるようキャンバスレンダリングを最適化する。
